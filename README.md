# WorkShare

WorkShare ist eine Android-App fuer Teams auf der Baustelle.

Mit WorkShare koennt ihr Projekte gemeinsam organisieren, Material live synchronisieren, Fotos teilen und Teammitglieder kontrolliert einladen.

## Schnellstart

### 1. App installieren
1. APK ueber euren internen Firmenlink herunterladen.
2. Auf Android installieren (bei Bedarf "Unbekannte Apps installieren" erlauben).
3. App starten und mit E-Mail + Passwort registrieren oder einloggen.

### 2. Erstes Projekt anlegen
1. In der Projektuebersicht auf `+` tippen.
2. Projektname eingeben und speichern.
3. Im Projekt Material, Notizen, Arbeitszeit und Fotos erfassen.

### 3. Team einladen
- Mitglieder koennen per E-Mail eingeladen werden.
- Alternativ koennen Nutzer per Projektcode beitreten.
- Einladungen erscheinen in der Glocke oben rechts.
- Jede Einladung kann angenommen oder abgelehnt werden.

## Hauptfunktionen

- Login und Registrierung (E-Mail/Passwort)
- Projekte anlegen, bearbeiten, abschliessen, loeschen
- Rollen im Projekt: `owner`, `admin`, `worker`, `viewer`
- Materialverwaltung mit Live-Synchronisation
- Materialkatalog mit Vorschlaegen beim Erfassen
- PDF-Export der Materialliste
- Fotos pro Projekt mit Cloud-Sync
- Workgroups fuer Teamarbeit
- Benachrichtigungen fuer Einladungen und Team-Aktionen
- Dark Mode, Light Mode und Systemmodus

## Benachrichtigungen

- Die Glocke zeigt neue Ereignisse (z. B. Einladungen, Beitritte, Entfernungen).
- Beim Oeffnen der Benachrichtigungen werden Eintraege als gelesen markiert.
- Neue Eintraege sind fett dargestellt.

## Updates

- In den Einstellungen kann man manuell auf Updates pruefen.
- Wenn ein Update verfuegbar ist, kann es direkt installiert werden.
- Update-Hinweise werden pro Version nur einmal angezeigt.

## Offline-Nutzung

- WorkShare funktioniert auch ohne Internet.
- Eingaben werden lokal zwischengespeichert.
- Sobald wieder Netz da ist, synchronisiert die App automatisch.

## Hinweise zur Teamnutzung

- Nur Projektmitglieder sehen Projektdaten.
- Beim Verlassen einer Workgroup bleiben Daten erhalten und werden bei Bedarf getrennt weitergefuehrt.
- Pro Benutzer ist nur eine aktive Workgroup gleichzeitig moeglich.
